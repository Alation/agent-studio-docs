Install the Alation plugin in Codex and work with your catalog in natural language.

## Prerequisites

- Python 3.10+ on the machine running the agent.
- An [OAuth client registered on your Alation instance](/agent-studio-docs/integrations/plugins/overview/#connect-to-your-alation-instance) (a one-time, per-instance step).

## Install

From the Codex **Plugins** page, click the **+** and choose **Add marketplace**.

<Image src={addMarketplaceMenu} alt="Codex Plugins page with the + menu open showing Create plugin and Add marketplace" style="max-width: 80%; height: auto;" />

Fill in the marketplace:

| Field | Value |
|-------|-------|
| Source | `https://github.com/Alation/alation-plugins` |
| Git ref | `main` |
| Sparse paths | `plugins/codex` |

<Image src={addMarketplaceModal} alt="Add plugin marketplace modal with the Alation source, main ref, and plugins/codex sparse path" style="max-width: 70%; height: auto;" />

Open the **Alation Plugins** tab and click **Add plugin** on the Alation entry.

<Image src={alationPluginAdd} alt="Alation Plugins tab showing the Alation plugin under Productivity with an Add plugin button" style="max-width: 100%; height: auto;" />

The plugin installs with all its skills and registers an Alation MCP bridge automatically.

<Image src={alationPluginDetail} alt="Alation plugin detail page listing the Ask, Automate, Configure, Curate, Explore, and Setup skills" style="max-width: 80%; height: auto;" />

<Aside type="tip" title="Install from the command line instead">
```bash
codex plugin marketplace add https://github.com/Alation/alation-plugins.git
codex plugin add alation@alation-plugins
```
</Aside>

## Authenticate

Ask Codex to set up the Alation plugin — this runs the `setup` skill.
It prompts for your instance **base URL** and the **Client ID** from the OAuth client, then uses a paste-the-redirect-URL login: open the authorization URL it gives you, log in, and paste the final redirect URL back into the chat.

<Aside type="note" title="The MCP bridge shares this login">
The Alation MCP bridge reuses the same credentials and OAuth token from setup and connects to `<base-url>/ai/mcp` — no separate MCP login.
If you finish setup after a thread has started, open a new thread so Codex relaunches the bridge with the new token.
</Aside>

You only [register the OAuth client](/agent-studio-docs/integrations/plugins/overview/#connect-to-your-alation-instance) once per instance.
