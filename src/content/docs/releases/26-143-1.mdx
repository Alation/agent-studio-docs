**Release Date:** 2026-06-10  

**Version Range:** v26.113.1 -> v26.143.1  

## Highlights  

- Enhanced data product schema with new fields for better context and usability.  
- Improved error handling and logging for metrics and agent operations.  
- Expanded chat message history limit to 1000, enabling deeper conversational context.  

## What's New  

### Features  

- **Data Product Enhancements:** You can now use the new `exampleQuestions` and `businessContext` fields in the data product schema to provide more context for your data.  
- **Chat History Expansion:** Chat message history limit has been increased to 1000, allowing for better continuity in conversations.  
- **Improved Lineage Tracking:** The system now prefers BI data sources for RDBMS-BI lineage and resolves upstream columns from any BI report for enhanced data lineage capabilities.  
- **Agent Configuration:** Added new features for managing agent batch summaries, including CRUD operations and batch retrieval endpoints.  
- **PII Sanitization:** A new tool has been introduced to sanitize personally identifiable information (PII) in user-pasted logs.  

### Improvements  

- **Metrics Logging:** Enhanced logging for multi-statement SQL blobs and improved error reporting for parse failures.  
- **Agent Performance:** Increased read timeout for Bedrock operations to ensure smoother processing during longer tasks.  
- **Lineage Efficiency:** Batch processing for report and data source column fetches improves performance and reduces processing time.  

### Bug Fixes  

- **Chat Privacy:** Chats are now private to their respective owners, ensuring secure communication.  
- **Error Handling:** Improved handling of transient mid-stream errors and actionable error messages for LLM operations.  
- **Configuration Fixes:** Adjusted read timeout settings for curation tasks and resolved issues with spaces in agent parameters.  
- **Metrics Parsing:** Addressed top-level parser errors and ensured better handling of SQL parse failures.  
- **Middleware Updates:** Fixed issues related to user ID acceptance and restored critical internal endpoints for infrastructure.  

### Breaking Changes  

None in this release.  

## Customer Impact

None in this release.
