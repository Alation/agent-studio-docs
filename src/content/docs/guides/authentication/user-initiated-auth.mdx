:::caution[Before you begin]
Only users with the **Server Admin** role can create OAuth clients in Alation.
:::

User (delegated) authorization refers to an OAuth flow where users grant third-party applications access to their protected resources.
In this case, the grant is to Alation.
When this occurs, the third-party application may act on behalf of the user within Alation.

Delegated authorization requires several systems interacting in concert and follows the **3-Legged Authorization Grant** flow.

:::tip[Learn more about OAuth flows]
- [OAuth 2.0 Concepts & OAuth Flows](https://iamblockc.medium.com/oauth-2-0-concepts-oauth-flows-8debed7a9abc) — Overview of different OAuth flows
- [IETF: OAuth Delegated Authorization](https://www.ietf.org/archive/id/draft-li-oauth-delegated-authorization-00.html) — Detailed specification
:::

For delegated authorization to work, configurations are required within Alation and within the third-party application.

:::caution
Throughout this guide, replace `your-alation-instance` with your specific Alation instance domain.
:::

## Creating an OAuth Client in Alation

You can navigate directly to the OAuth Client Applications page at:
```txt "your-alation-instance"
https://your-alation-instance.alationcloud.com/app/admin/oauth-client-apps/
```

Or follow the steps below:

<Steps>
1. **Open Admin Settings**

   Click the **gear icon** in the top-right corner of the Alation navigation bar to open the Admin Settings page.

   <Image src={settingsGearIcon} alt="Gear icon in the Alation top navigation bar showing the Settings tooltip" style="max-width: 50%; height: auto;" />

2. **Navigate to OAuth Client Applications**

   Under the **User Management** section, click the **OAuth Client Applications** card.

   <Image src={adminSettingsPage} alt="Admin Settings page showing the User Management section with the OAuth Client Applications card" style="max-width: 80%; height: auto;" />

3. **Fill out the creation form**

   Click **Create** to open the form. Fill in the following fields:

   - **Name** — A unique display name for your OAuth client (e.g., the platform or application it's for).
   - **Client Type** — Select **Confidential** (recommended) for backend integrations where credentials can be kept secure, or **Public** for single-page applications or mobile apps.
   - **Redirect URIs** — The callback URI(s) where Alation will redirect users after authorization. Enter one URI per line. HTTP is only allowed for `localhost` or `127.0.0.1` (development). See [Known Redirect URIs](#known-redirect-uris) below.
   - **Access Token Expiry (seconds)** — Lifetime of access tokens. Min: 3,600 (1 hour), Max: 259,200 (3 days).
   - **Refresh Token Expiry (seconds)** — Lifetime of refresh tokens. Min: 259,200 (3 days), Max: 15,552,000 (180 days).
   - **PKCE Required** — Whether to enforce Proof Key for Code Exchange. Recommended for security. Must be enabled for public clients. See [PKCE notes](#pkce-proof-key-for-code-exchange) below.

   Click **Continue** when done.

   <Image src={createOAuthForm} alt="Create User-Initiated OAuth Client form with fields for name, client type, redirect URIs, token expiry, and PKCE" style="max-width: 70%; height: auto;" />

4. **Copy your Client ID and Client Secret**

   After creation, a success dialog displays your **Client ID** and **Client Secret**. Use the **Copy** buttons to save both values.

   :::caution[Important]
   The **Client Secret** is only shown once. Copy it before closing this dialog — it cannot be retrieved later.
   :::

   <Image src={creationSuccess} alt="Success dialog showing the Client ID and Client Secret with copy buttons" style="max-width: 70%; height: auto;" />
</Steps>

## Managing Existing OAuth Clients

Each OAuth client in the list has a three-dot menu (**...**) with the following actions:

- **View / Edit** — View or modify the client's settings (name, redirect URIs, token expiry, PKCE). Note that the **Client Secret is not viewable** from this screen — if you've lost it, you must rotate it or create a new client.
- **Rotate Secret** — Generate a new client secret. The old secret is immediately invalidated, and the new one is shown only once.
- **Delete** — Permanently delete the OAuth client.

<Image src={contextMenu} alt="Context menu on an OAuth client row showing View/Edit, Rotate Secret, and Delete options" style="max-width: 30%; height: auto;" />

## Third-Party Application Configuration

To connect your third-party application to Alation, configure it with the following values:

**Authorization URL:**
```txt "your-alation-instance"
https://your-alation-instance.alationcloud.com/oauth/v1/authorize/
```

**Token URL:**
```txt "your-alation-instance"
https://your-alation-instance.alationcloud.com/oauth/v1/token/
```

You'll also need the **Client ID** and **Client Secret** that were generated when you [created the OAuth client](#creating-an-oauth-client-in-alation).

### Platform-Specific Considerations

Different platforms have different requirements and capabilities.

:::tip[Platform-specific setup guides]
For detailed step-by-step instructions for each platform, refer to their individual documentation pages in the [MCP Client Setup guides](../../mcp-client-setup/introduction/).
:::

### Known Redirect URIs

- **VSCode:**
  ```
  https://vscode.dev/redirect
  ```

- **Claude Desktop:**
  ```
  https://claude.com/api/mcp/auth_callback
  ```
  ```
  https://claude.ai/api/mcp/auth_callback
  ```

- **ChatGPT Connectors:**
  ```
  https://chatgpt.com/connector_platform_oauth_redirect
  ```

- **Cursor:**
  ```
  cursor://anysphere.cursor-mcp/oauth/callback
  ```

:::tip[MS Copilot Studio configuration]
If you're creating a configuration for MS Copilot Studio, you'll need to:
1. Create the OAuth client without the Redirect URI first
2. Configure the Client ID and Client Secret within Copilot Studio
3. Obtain the Redirect URI that Copilot Studio generates
4. Edit the OAuth client in Alation to add that Redirect URI
:::

### PKCE (Proof Key for Code Exchange)

PKCE is an additional security measure added to OAuth 2.0.
Not all platforms support it — if you're in doubt, it's best to leave it enabled as a starting point.

:::caution[PKCE Support]
ChatGPT and MS Copilot Studio do not yet support PKCE.
Disable the **PKCE Required** option when creating OAuth clients for those platforms.
:::
