Agent Studio lets you build AI agents grounded in the trusted context already in Alation: your catalog, your data products, and your databases.

Enterprise agents usually fail for one reason: they lack the right context.
Knowledge is fragmented across systems, definitions conflict or go stale, and keeping agents accurate by hand doesn't scale.
Agent Studio fixes this by grounding every agent in the [Alation Knowledge Layer](/agent-studio-docs/get-started/key-concepts/#what-is-the-alation-knowledge-layer), the continuously maintained metadata your organization already trusts.

Inside Alation, the Agent Studio web UI is where you create, test, evaluate, and publish — no code required.
Outside Alation, the same tools and agents are available over [MCP](/agent-studio-docs/integrations/mcp/quick-start/), the [REST API](/agent-studio-docs/integrations/rest-api/auth/), and the [Python SDK](/agent-studio-docs/integrations/sdk/using-the-sdk/), so you don't have to move your agent platform to benefit from its context.

## Who is it for?

Everyone is a builder, but the entry points differ.
Business analysts run and customize agents in the UI.
Data analysts and stewards iterate in the UI and operationalize over MCP or the SDK.
AI developers consume tools and agents from MCP, REST, and Python in their existing platforms.

<Aside type="note" title="Enabling Agent Studio">
An Alation admin can turn it on under **Settings > Feature Configuration**: toggle Agent Studio on and click **Save**.
It appears in the left icon rail of the New User Experience.
</Aside>
